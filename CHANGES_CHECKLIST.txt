7. API ENDPOINTS
================================================================================

✅ POST /api/track-error
   - Parameter: message
   - Persists to Redis with UUID

✅ GET /api/tracked-stacktraces
   - Retrieves all traces from Redis
   - Returns count and list

✅ GET /api/tracked-stacktraces/ip/{ip}
   - Filters traces by IP from Redis
   - Uses Redis IP index

✅ DELETE /api/clear-stacktraces
   - Clears all data from Redis
   - Clears all indexes

✅ GET /api/health
   - Health check endpoint
   - Returns service status

✅ GET /api/documentation
   - HTML documentation page

================================================================================
8. REDIS DATA STRUCTURE
================================================================================

✅ Individual Traces
   Key format: stacktrace:{uuid}
   Value: JSON-serialized StackTraceData object

✅ Trace Index
   Key: stacktraces:index
   Type: Redis Set
   Members: All trace UUIDs

✅ IP Index
   Key format: stacktraces:ip:{ip_address}
   Type: Redis Set
   Members: Trace UUIDs for that IP
