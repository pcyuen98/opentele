🔍 OpenTelemetry Stacktrace Tracker

Track, monitor, and analyze application errors with OpenTelemetry and Spring Boot 4.0

📋 Overview

The OpenTelemetry Stacktrace Tracker is a Spring Boot 4.0 application built with Java SDK 21 that provides comprehensive error tracking and monitoring capabilities. It captures stack traces along with essential metadata including source IP address, timestamp, and date of occurrence.

🌐 IP Tracking

Automatically captures the source IP address of error occurrences, supporting X-Forwarded-For and X-Real-IP headers.

⏰ Timestamp Tracking

Records precise timestamps for when errors occur, enabling time-based analysis.

📅 Date Tracking

Automatically extracts and stores the date of error occurrences for easy date-based filtering.

📊 OpenTelemetry Integration

Leverages OpenTelemetry for distributed tracing and monitoring capabilities.

🚀 Getting Started

Prerequisites

  • Java SDK 21 - Required for compilation and runtime
  • Maven 3.8.0+ - For building the application
  • Spring Boot 4.0 - Application framework

Installation & Setup

  1. Build the Application
    mvn clean install
  2. Start the Spring Boot Application
    mvn spring-boot:run
    ✅ The application will start on http://localhost:8080
  3. Verify the Application is Running
    curl http://localhost:8080/api/health

📡 API Documentation

1. Health Check Endpoint

GET
/api/health

Description: Check if the application is running and operational

Parameters: None

Example Response (200 OK)

{ "status": "UP", "message": "OpenTelemetry Stacktrace Tracker is running" }

2. Track Error Endpoint

POST
/api/track-error

Description: Track an error with stacktrace, IP, timestamp, and date

Parameters:

Parameter Type Required Description
message String Yes The error message to track

Example Request

curl -X POST "http://localhost:8080/api/track-error?message=Database connection failed"

Example Response (200 OK)

{ "status": "tracked", "ip": "127.0.0.1", "timestamp": "2024-06-14T10:21:27.437+00:00", "date": "2024-06-14", "errorMessage": "Database connection failed", "errorType": "java.lang.Exception", "stackTraceLength": 12 }

3. Get All Tracked Stacktraces

GET
/api/tracked-stacktraces

Description: Retrieve all tracked stacktraces with their metadata

Parameters: None

Example Response (200 OK)

{ "count": 2, "stacktraces": [ { "ip": "127.0.0.1", "timestamp": "2024-06-14T10:21:27.437+00:00", "date": "2024-06-14", "errorMessage": "Database connection failed", "errorType": "java.lang.Exception", "stackTrace": [ "com.opentele.stacktrace.StackTraceController.trackError(...)", "..." ] }, { "ip": "192.168.1.100", "timestamp": "2024-06-14T10:22:15.123+00:00", "date": "2024-06-14", "errorMessage": "Null pointer exception", "errorType": "java.lang.Exception", "stackTrace": [...] } ] }

4. Get Stacktraces by IP Address

GET
/api/tracked-stacktraces/ip/{ip}

Description: Retrieve all stacktraces from a specific IP address

Path Parameters:

Parameter Type Description
ip String The IP address to filter by

Example Request

curl "http://localhost:8080/api/tracked-stacktraces/ip/192.168.1.100"

Example Response (200 OK)

{ "ip": "192.168.1.100", "stacktraces": [ { "ip": "192.168.1.100", "timestamp": "2024-06-14T10:22:15.123+00:00", "date": "2024-06-14", "errorMessage": "Null pointer exception", "errorType": "java.lang.Exception", "stackTrace": [...] } ] }

5. Clear All Tracked Stacktraces

DELETE
/api/clear-stacktraces

Description: Clear all tracked stacktraces from memory

Parameters: None

Example Request

curl -X DELETE "http://localhost:8080/api/clear-stacktraces"

Example Response (200 OK)

{ "status": "cleared", "message": "All tracked stacktraces have been cleared" }

✨ Key Features

  • Automatic IP Extraction: Captures client IP address from requests, supporting X-Forwarded-For and X-Real-IP headers for proxy environments
  • Precise Timestamp Recording: Records exact time when errors occur using LocalDateTime
  • Date Tracking: Automatically extracts and formats date information for easy filtering
  • Complete Stacktrace Capture: Captures full Java stacktrace elements with class, method, and line number information
  • OpenTelemetry Integration: Uses OpenTelemetry for distributed tracing and span creation
  • In-Memory Storage: Fast access to tracked stacktraces with list-based storage
  • IP-Based Filtering: Query stacktraces by source IP address
  • REST API: Easy-to-use REST endpoints for all operations

📦 Data Model

StackTraceData Object

Represents a single tracked error with the following properties:

Property Type Description
ip String Source IP address of the error (extracted from request headers)
timestamp LocalDateTime Precise timestamp when error was tracked
date String Date in ISO_LOCAL_DATE format (YYYY-MM-DD)
stackTrace String[] Array of stacktrace elements showing the call sequence
errorMessage String The error message text
errorType String Fully qualified exception class name

🏗️ Architecture

Component Overview

  • Application.java: Spring Boot application entry point
  • OpenTelemetryConfig.java: Configures OpenTelemetry SDK with logging span exporter
  • StackTraceTrackerService.java: Business logic for tracking and storing stacktraces
  • StackTraceController.java: REST API endpoints for stacktrace operations
  • StackTraceData.java: Data model representing tracked stacktrace information

Data Flow

  1. HTTP request arrives at StackTraceController
  2. Controller extracts client IP address from request headers
  3. Service creates StackTraceData with IP, timestamp, and date
  4. OpenTelemetry span is created with error metadata
  5. Stacktrace is stored in in-memory list
  6. Response is returned with tracked data details

⚙️ Configuration

Application Properties

The application can be configured via application.properties:

Property Default Value Description
spring.application.name opentele-stacktrace Application identifier
server.port 8080 Port on which application runs
logging.level.root INFO Root logger level
logging.level.com.opentele.stacktrace DEBUG Application package logger level

Modify Port

To run on a different port, update application.properties:

server.port=9090

Then restart the application.

🔧 Troubleshooting

Build Fails

Issue: Maven build fails with compilation error
Solution: Ensure Java 21 is installed and set as JAVA_HOME
java -version export JAVA_HOME=/path/to/java21 mvn clean install

Application Won't Start

Issue: Port 8080 already in use
Solution: Change port in application.properties or kill the process using port 8080
# Find process using port 8080 lsof -i :8080 # Kill the process kill -9

API Endpoints Not Responding

Issue: Endpoints return 404 error
Solution: Verify application is running and check endpoint URL
# Verify health endpoint curl -v http://localhost:8080/api/health # Check logs for errors tail -f application.log

💡 Best Practices

  • Regular Cleanup: Periodically clear tracked stacktraces to prevent memory buildup. Consider using the DELETE endpoint or implementing scheduled cleanup.
  • IP Header Configuration: In production, ensure your load balancer or proxy correctly sets X-Forwarded-For or X-Real-IP headers for accurate IP tracking.
  • Date-Based Analysis: Use the date field for aggregating errors by day for reporting and trend analysis.
  • Error Monitoring: Regularly check the health endpoint to ensure the application is operational.
  • Timestamp Precision: Timestamps are recorded with millisecond precision using LocalDateTime. This ensures you can identify rapid error sequences.
  • Production Deployment: Consider connecting OpenTelemetry to a monitoring backend like Prometheus or Jaeger for production use.
