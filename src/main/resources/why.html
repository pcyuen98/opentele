Root cause: Redis persistence not invoked because redisService is null

Evidence from code (file paths & key excerpts):

  • StackTraceTrackerService (src/main/java/com/opentele/stacktrace/service/StackTraceTrackerService.java): 
          @Setter
      private RedisStackTracePersistenceService redisService;

      public StackTraceTrackerService() {
        this.tracer = GlobalOpenTelemetry.getTracer("stacktrace-tracker");
        this.trackedStackTraces = new ArrayList<>();
        this.redisService = null;  // explicitly set to null
      }

      ...

      if (redisService != null) {
        redisService.persistStackTrace(traceId, data);
      }
  • RedisStackTracePersistenceService exists and is a @Service (src/main/java/com/opentele/stacktrace/persistence/RedisStackTracePersistenceService.java) and would persist data when called (persistStackTrace writes keys and index sets).

Why this prevents persistence

Because StackTraceTrackerService never receives a RedisStackTracePersistenceService instance at runtime (redisService remains null), the condition (redisService != null) is false and persistStackTrace is never invoked. Thus no data is written to Redis despite Redis beans and configuration being present (application.properties, RedisConfig).

Secondary note

The controller method trackError has only the first parameter annotated with @RequestParam; the other String parameters (errorCode, stacktrace) are un-annotated which can lead to null values coming from requests — this is separate and would not by itself stop persistence but may result in incomplete stored data.

How to fix (non-exhaustive, no code changes applied here)

  • Inject RedisStackTracePersistenceService into StackTraceTrackerService. Preferred: constructor injection (make the dependency final and include it in constructor).
  • Or annotate the setter with @Autowired so Spring calls the setter and sets redisService.
  • Alternatively, remove the explicit no-arg constructor and let Lombok/@RequiredArgsConstructor generate a constructor that includes required beans for injection.
  • Add @RequestParam to the other controller params (errorCode, stacktrace) to ensure values are bound from request parameters.
